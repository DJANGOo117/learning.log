# AGENTS.md

## 协作规则

1. 每次让 Codex 改代码前，先要求它说明会改哪些文件、为什么。
2. 每次改完代码后，Codex 要自己在浏览器点一遍相关功能。
3. 每天结束时运行一次 `git status`，确认没有意外文件。
4. 不把密码、API Key、Cookie、个人隐私数据发给 Agent，也不提交到 GitHub。

## 编码规则

1. 编码前先思考：假设要说出来，不确定就问；有多种理解时，列出来让人选；有更简单的方案，主动说明；搞不清楚就停下来，不要蒙头干。
2. 简约至上：只写刚好够用的代码，不加没要求的功能，不做单次用的抽象封装。
3. 精确编辑：只动要动的文件，不顺手改周边代码，不重构没坏的东西，沿用现有风格；发现无关死代码只提醒，不删除。
4. 目标驱动执行：把任务转成可验证的成功标准；修 bug 要写能复现的测试并让它通过；多步任务先列计划和验证方式。
